# github contribution activity 
제대로 기입되지 않은 활동 내역을 다시 기입하는 방법을 기술해두었음

# 주의할점?
커밋하지 않은 날짜에 커밋을 한 것 처럼 위장하는것은 불가능함
커밋을 했던 파일 중 깃허브 활동에 기록되지 않은 것들에 한해서 기록 가능


# 원인 
원인은 다양하나 주로 깃의 이메일과 깃허브의 이메일이 다를 때 발생함



# 해결 방안 -----------------------------------------------------------

1. 이메일 주소가 올바른지 확인
(깃허브에 연동되어있는 이메일과 같은지?)
git config --list


2. 이메일 주소가 올바르지 않을때
    로컬 전체에 이메일 변경 적용
    git config --global user.email "변경할 이메일 주소"
    작업하고 있는 파일에만 적용
    git config user.email "변경할 이메일 주소"

 
3. 기록되지 않은 활동의 해쉬코드 찾기
git log --pretty=format:"%h = %an, %ar : %s" --graph
출력된 로그 중 기록되지 않은 commit의 앞쪽에 적혀있는 해쉬코드 기억해야함


4. 앞에서 봐둔 해쉬코드로 rebase
git rebase -i -p 해쉬코드
출력된 vim 을 수정모드로 변경하고, 화면에 나타난 커밋들 중에서 
변경하고자 하는 커밋들만 골라 가장 앞쪽에 적혀있는 pick을 edit으로 수정


5. 수정한 파일을 저장하고 vim을 종료


6. commit 할 때 사용할 이메일 수정
git commit --amend --author="이름 <커밋할 이메일>"
앞쪽에 새롭게 수정한 이메일로 재커밋 할 것 이기 때문에
수정한 이메일을 넣어 커밋하였음.
입력하고 나면 다시 vim이 나타나는데, 재커밋만 할 것이므로 종료해줌


7. git rebase --continue명령으로 남은 rebase를 계속함


8. 남은 rebase의 수(pick을 edit으로 수정한 수)만큼 6번과 7번 항목 반복


9. 원격 저장소에 커밋 강제 푸시
git push origin +브랜치명
마스터에 바로 커밋할것이므로 +master를 사용했음


10. 깃허브에 접속하면 실수로 기록되지 않거나 사라져버린  기록들이 복원되어있음
